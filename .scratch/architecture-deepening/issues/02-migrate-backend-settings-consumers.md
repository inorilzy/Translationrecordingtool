# 02 — 迁移后端设置调用方到 canonical record

**What to build:** 让启动装配、翻译工作流、快捷键、托盘、主题和 OCR 配置都从同一个 canonical settings record 读取和持久化，使用户更新设置后所有入口立即使用同一份运行时值。

**Blocked by:** 01 — 扩展 canonical settings runtime seam

**Status:** ready-for-agent

- [ ] 应用启动时直接从 canonical snapshot 装配运行时状态，不再依赖逐字段复制的完整设置镜像。
- [ ] 用户更新供应商或凭证后，手动输入、剪贴板、选中文本快捷键和截图翻译的下一次请求都会读取新值。
- [ ] 全局快捷键、截图快捷键、托盘行为与主题更新仍能立即生效并在重启后恢复。
- [ ] OCR 模型 profile、预热选项及当前兼容字段继续按现有行为保存和读取。
- [ ] 没有生产后端调用方继续以旧完整运行时设置记录作为权威来源。
- [ ] 测试通过可观察的设置快照、工作流选择和重启持久化结果验证行为，不断言内部锁或字段赋值顺序。
